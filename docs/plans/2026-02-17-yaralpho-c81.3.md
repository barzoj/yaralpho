# yaralpho-c81.3 Verification Timeout Implementation Plan

After _human approval_, use plan2beads to convert this plan to a beads epic, then use `superpowers:subagent-driven-development` for parallel execution.

**Goal:** Enforce a dedicated verification timeout so verification runs stop promptly, record timed-out TaskRuns, and surface errors for retry/failure handling.

**Architecture:** Add a verification-context timeout in the worker’s verification path using the configured `YARALPHO_TASK_VERIFY_TIMEOUT`. The verification task should inherit the timeout context so `executeTask` marks runs timed out and Stop hooks fire on deadline. Tests will cover timeout behavior and status/error propagation.

**Tech Stack:** Go 1.22, Go stdlib `context`, existing consumer/worker/verification task code, Go testing (`testing`, `require`).

**Key Decisions:**

- **Timeout source:** Use the dedicated `YARALPHO_TASK_VERIFY_TIMEOUT` config to bound verification duration — mirrors execution timeout but keeps independent control per acceptance criteria.
- **Context application:** Wrap verification in a child context with `context.WithTimeout` so `executeTask` sets `TaskRunStatusTimedOut` and defers Stop on deadline — ensures Stop hook always runs and status updates remain centralized.
- **Error propagation:** Return verification timeout errors from `executeAndVerify` to the worker loop — aligns with existing retry/failure handling and surfaces timeouts consistently.
