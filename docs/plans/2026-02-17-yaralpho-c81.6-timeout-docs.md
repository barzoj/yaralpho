# yaralpho-c81.6 Timeout Docs Implementation Plan

After _human approval_, use plan2beads to convert this plan to a beads epic, then use `superpowers:subagent-driven-development` for parallel execution.

**Goal:** Document execute/verify 20m defaults, how to override them, and how timeouts affect retries/agent release; refresh configuration examples accordingly.

**Architecture:** Documentation-only change: consolidate timeout defaults/override guidance in README, align JSON example, and ensure design doc notes scheduler handling. No code paths are modified; docs must mirror current behavior (timeouts treated as failures that free agents and increment attempts).

**Tech Stack:** Markdown docs, JSON config example.

**Key Decisions:**

- **Single-source defaults:** Keep README as the authoritative timeout reference, echoed in config.example.json — avoids drift across docs.
- **Override priority messaging:** Emphasize env-over-JSON precedence — aligns with existing config loader behavior and reduces misconfiguration risk.
- **Behavior framing:** Describe timeouts as failure-equivalent for retries/agent release — matches scheduler semantics and sets expectations for operators.
